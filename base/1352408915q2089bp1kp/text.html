1. Содержание файла /etc/apt/sources.list должно быть примерно таким (перед изменениями лучше сделайте backup старого файла):

###testing

deb http://security.debian.org/ testing/updates main contrib non-free

deb-src http://security.debian.org/ testing/updates main contrib non-free

deb http://ftp.ru.debian.org/ testing main contrib non-free

deb-src http://ftp.ru.debian.org/ testing main contrib non-free

deb http://ftp.ru.debian.org/ testing-proposed-updates main contrib non-free

deb-src http://ftp.ru.debian.org/ testing-proposed-updates main contrib non-free

2. Затем выполните от администратора команды:

aptitude update

aptitude safe-upgrade

aptitude dist-upgrade

aptitude safe-upgrade - приведёт к обновлению только тех пакетов, которые можно обновить без необходимости удаления или установки других пакетов.

aptitude dist-upgrade - произведёт полное обновление системы, т.е. установку последних доступных версий всех пакетов и разрешение всех изменившихся между выпусками зависимостей. При необходимости будут установлены новые пакеты (обычно, это новые версии библиотек или переименованные пакеты) и удалены все вызывающие конфликты устаревшие пакеты.

0x0AB411C0 пишет:

Где-то встречал, что перед "Обновляешь систему полностью или только избранные пакеты." иммет смысл делать

aptitude install aptitude dpkg apt-get

Еще перед обновлением неплохо сделать

aptitude update && aptitude -f install

Проверит все ли в порядке с пакетами и нет ли нерешенных зависимостей.

Что по команде из вашего поста:

# aptitude install aptitude

По этой команде автоматически обновятся пакеты libc6 и locales, и установятся библиотеки поддержки SELinux (libselinux1). После этого будут перезапущены некоторые работающие службы, включая xdm, gdm и kdm, а следовательно, могут прерваться локальные сеансы X11.
