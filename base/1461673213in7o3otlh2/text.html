Имеем Oracle Primavera P6 EPPM 8.2 и Microsoft Project 2010. В какой-то момент перестает работать импорт из Project. Какими средствами можно восстановить работу импорта? Компания Oracle оставила наш запрос без внимания.


Временно работающие и неработающие решения следующие:


  • Перезагрузить сервер баз данных - не помогает.
  • Попробовать сделать импорт через Web-интерфейс (а не через Java-интерфейс) - сработало один раз.
  • Попробовать удалить шаблон импорта и создать новый шаблон в Java-интерфесе - сработало один раз.
  • Полная перестановка сервера - не пробовали и пробовать не будем, там с ума сойти можно.


Буржуи пишут, что есть еще один вариант. Есть сторонняя программа Primavera P6 EPPM Administration Tool, загадочной компании EPPMSoft Inc. Описание этой программы здесь:


http://www.eppmsoft.com/articles/p6eppm_admin/P6EPPM_Admin_doc.php


Стоимость программы на сайте не указана. Распространяется 30-ти дневная триальная версия. В ней есть конвертор из MPP в XER файл:



Самое интересное, что для запуска конвертации не нужно подключение к серверу. Не нужно вносить параметры подключения в файл runP6EPPMAdmin.cmd, как того требует документация. Не нужно даже устанавливать программу. Достаточно просто наличия Java на компьютере и прописывания пути к каталогу /bin где лежит java.exe в системную переменную PATH. Или можно прописать этот путь в переменную JAVA_HOME в самом runP6EPPMAdmin.cmd. После этого функции конвентирования просто работают, и, кстати, не видно никакого программного ограничения на 30 дней.
