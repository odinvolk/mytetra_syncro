Нашел такое решение для себя.

использую утилитку psexec и батник с сообщением, батник потому что проблемы с кодировкой при передаче кириллицы через psexec

1. cmd - > psexec \\<имя компа/ip адресс> Msg * <текст сообщения> - psexec зайдет на комп и выполнит команду Msg, сообщение увидит залогиненый пользователь. (но кириллица у меня не работает)

2. для отправки кирилицы psexec \\<имя компа/ip адресс> -i -s -c <путь к батнику на вашем компе>

в батнике пишем Msg * <текст сообщения> - уже можно на русском. psexec скопирует батник на удаленный комп и выполнит его оттуда и проблем с кодировкой не будет.

Чтобы отправить конкретному юзеру, нужно найти где он залогинен. для этого можно при событии logon запускать батник, и создавать типа базу данных кто куда входит

echo logon %computername% %date% %time% >> \\путь куда ложить логи\%username%.txt

открываем лог нужного пользователя и видим на какой комп слать мессадж.
