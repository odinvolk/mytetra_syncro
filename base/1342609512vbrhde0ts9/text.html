Установка сервера терминалов XRDP попутно решение проблем с кодировкой

Ставим сервер терминалов XRDP

yum install tigervnc-server

yum install xrdp

теперь разрешим в iptables коннектится клиентам по порту 3389

nano /etc/sysconfig/iptables

добавим

# RDP connection port

-A INPUT -p tcp -m state -m tcp --dport 3389 --state NEW -j ACCEPT

перезапуск фаервола

/etc/init.d/iptables restart

запуск XRDP сервера

/etc/init.d/xrdp start

Заходим клиентом mstsc в windows, rdesktop в linux. Если в Linux нет такого, то устанавливаем:

yum install rdesktop

Но терминальная сессия открывается в английской локали - Gnome английский, в терминале вместо русских шрифтов ????.

хотя команда locale вам скажет, что все в порядке вы локализованы ru_Ru.UTF-8

Это лечится, если вы работали с Ubuntu в ее настройках есть лекарство для xrdp.

В /etc/xrdp/startwm.sh добавить

if [ -r /etc/default/locale ]; then

. /etc/default/locale

export LANG LANGUAGE

fi

если еще нет, создать файл /etc/default/locale с содержимым:

LANG="ru_RU.UTF-8"

сделали ? Перезаходим — все по русски.

в файле /etc/xrdp/startwm.sh за коментами есть подсказка, как настроить переменную PATH для терминальной сессии

Проверено на Fedora13.
