Если при монтировании удаленного ресурса по сети возникает подобная ошибка:


mount /media/mycloud

mount: wrong fs type, bad option, bad superblock on 192.168.1.9:/nfs/Public,

      missing codepage or helper program, or other error

      (for several filesystems (e.g. nfs, cifs) you might

      need a /sbin/mount.<type> helper program)


      In some cases useful info is found in syslog - try

      dmesg | tail or so.


... то это значит, что скорее всего в системе не установлены соответствующие mount-пакеты для данных сетевых файловых систем. Обычно проблема решается установкой пакетов:


apt-get install cifs-utils nfs-common


После чего монтироване начинает работать.
