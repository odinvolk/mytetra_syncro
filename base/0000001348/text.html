Директивы основного блока

Директива user

Используется для указания, от какого пользователем будут запускаться рабочие процессы (так называемые веркеры). Сервер nginx всегда запускает основной процесс-обозреватель, который контролирует работу рабочих процессов-веркеров. Процесс-обозреватель запускается от пользователя root в момент старта сервера. Он порождает процессы-веркеры, которые будут запущены от имени пользователя, указанного в директиве user.

Пример:

user www-data;

Следует отметить, что этот параметр обычно должен совпадать с пользователем, установленным на файлы и каталоги, расположенные в директории вебсервера.

Директива worker_processes

Указывает, сколько процессов-веркеров должен запустить nginx. Для небольших сайтов вполне достаточно трех процессов. Пример:

worker_processes 3;

Директива error_log

Указывает файл, в котором размещается лог ошибок сервера nginx. Пример:

error_log /var/log/nginx/error.log;

Директива pid

Указывает файл, в котором будет храниться PID - номер запущенного основного процесса-обозревателя. Пример:

pid /var/run/nginx.pid;
